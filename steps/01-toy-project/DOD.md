# Definition of Done — Этап 1

Этап завершён, когда джун **понимает и может объяснить** все пункты, а прототип демонстрирует описанное поведение. Ментор отмечает каждый пункт.

## Модель диалога

- [ ] Джун объясняет, зачем агенту три роли сообщений: user, assistant, tool
- [ ] Джун объясняет, чем сообщение ассистента «хочу вызвать tool» отличается от сообщения tool «вот результат»
- [ ] Джун понимает, что история сообщений — это контекст для следующего решения модели
- [ ] Прототип сохраняет полную историю диалога за run

## Agent Loop

- [ ] Джун может описать цикл своими словами: решение → действие → обновление контекста → повтор
- [ ] Прототип на каждом шаге спрашивает модель «что дальше?»
- [ ] Если модель просит tool — tool выполняется, результат попадает в историю
- [ ] Если модель даёт финальный ответ — run завершается
- [ ] При достижении лимита шагов run останавливается со статусом max_steps, не падая

## Tool Call

- [ ] Джун объясняет: tool call — это данные (имя + аргументы), а не немедленное действие
- [ ] Джун объясняет разницу между «модель решила вызвать tool» и «runtime выполнил tool»
- [ ] Прототип имеет минимум 2–3 mock-инструмента с понятным поведением
- [ ] Unknown tool возвращает ошибку в контекст, run продолжается

## Mock Model

- [ ] Джун объясняет, зачем mock нужен для тестов и edge cases
- [ ] Mock возвращает предсказуемые ответы (не random, не network)
- [ ] Автотесты покрывают: happy path, unknown tool, max steps — только на mock

## Real Provider (OpenAI-compatible)

- [ ] Подключён один провайдер с OpenAI-compatible chat API (OpenAI / OpenRouter / локальный endpoint)
- [ ] Base URL, API key и model name захардкожены в коде — осознанно, без env на этом этапе
- [ ] В live-режиме модель сама выбирает tool call или final answer (без скрипта)
- [ ] Описание доступных tools передаётся в API вместе с историей сообщений
- [ ] Timeout на HTTP-запрос; при ошибке сети / 401 / 429 run → status failed, не зависает
- [ ] Переключение mock / live одним флагом; Agent Loop не меняется между режимами
- [ ] Live-прогон с простой задачей (calculator) успешно продемонстрирован ментору
- [ ] Джун объясняет: почему live не пишем в pytest (недетерминизм, cost, network)

## Run Lifecycle

- [ ] Джун перечисляет возможные финальные статусы: completed, failed, max_steps
- [ ] Джун объясняет, зачем нужен лимит шагов и что будет без него
- [ ] Прототип явно показывает текущий статус и количество пройденных шагов

## Наблюдаемость

- [ ] На каждом шаге видно: номер шага, что решила модель, результат или ошибка
- [ ] Финальный ответ пользователю отделён от промежуточных tool-шагов
- [ ] По логу run можно восстановить, что агент делал и почему

## Понимание (устная проверка)

- [ ] «Чем агент отличается от chatbot?» — через цикл и tool calls
- [ ] «Что произойдёт, если tool упадёт?» — ошибка в контекст, не крах run
- [ ] «Что изменится на этапе 2?» — ModelClient Protocol, env вместо hardcode
- [ ] «Зачем mock, если есть API?» — детерминизм, скорость, cost, тесты без сети
- [ ] «Почему hardcode ключ на этапе 1?» — минимум отвлечений; на этапе 2+ env
